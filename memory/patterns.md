## 错误处理与持续改进模式

### 模式 1: 错误承认与举一反三
- **原则**: 遇到错误不回避，承认→改正→举一反三
- **操作**:
  1. 第一时间承认错误，不掩饰
  2. 立即采取改正措施
  3. 分析根因，识别类似隐患
  4. 更新工作模式避免重复

### 模式 2: 失败深度分析
- **原则**: 任务失败后必须深入分析，不轻易放过
- **流程**:
  1. 立即排查问题所在
  2. **自身问题** → 能自动修复的及时修复
  3. **外部因素** → 找到根因，尽力修复
  4. **无法修复** → 记录详情，积累为经验
- **记录**: 所有失败案例写入 `patterns.md` 或 `feedback.md`

### 模式 3: 经验持续积累
- **原则**: 每次失败都是学习机会，不白白浪费
- **方法**:
  - 建立"失败案例库"
  - 定期回顾总结
  - 提炼通用解决方案

## AI 编程模式

### 模式 1: AI 辅助开发工作流
- **场景**: 使用 AI 工具（Cursor, Claude Code 等）开发软件
- **模式**: 
  1. 用 AI 生成初始代码框架
  2. 人工审查和调整架构
  3. AI 协助实现细节
  4. 自动化测试验证
- **适用**: 新功能开发、代码重构
- **注意事项**: 需人工把控关键设计决策

## 记忆管理模式

### 模式 1: 重要性评分
- **场景**: 决定是否记录某条信息
- **评分标准**:
  - 5分: 核心身份、重要决策、关键联系人
  - 4分: 项目信息、重要偏好、有效模式
  - 3分: 有用的背景信息、临时项目状态
  - 2分: 一次性信息、短期有效
  - 1分: 闲聊、临时上下文
- **操作**:
  - ≥4分 → 写入长期记忆
  - 2-3分 → 写入当日日志
  - <2分 → 不记录

### 模式 2: 信息提炼
- **场景**: 从每日日志提取长期价值信息
- **步骤**:
  1. 识别重复出现的主题
  2. 提取通用模式或解决方案
  3. 更新到对应专题文件
  4. 清理过时日志
