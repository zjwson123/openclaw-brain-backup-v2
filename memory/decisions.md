- **日期**: 2026-03-12
- **背景**: 需要建立一套结构化的记忆系统来管理长期和短期信息
- **选项**: 
  - 单层记忆（只有 MEMORY.md）
  - 双层记忆（长期 + 短期）
  - 复杂数据库方案
- **决策**: 采用双层记忆结构（长期专题文件 + 短期每日日志）
- **理由**: 
  - 平衡了简单性和结构化
  - 便于人工查看和编辑
  - 与 OpenClaw 原生机制兼容
  - 支持时间衰减策略
- **预期结果**: 更精准的上下文管理，减少重复沟通
