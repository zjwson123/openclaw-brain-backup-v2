# 技能积累机制 (Learning-from-Failure System)

## 核心理念
受傅盛"养龙虾"启发：Agent 不应是一次性工具，而应在使用过程中持续学习、积累 Skills。

> "养的过程中，不断地丰富、壮大自己的能力，形成很多的 skills"

## 触发条件
当以下场景发生时，主动触发技能沉淀：

1. **试错成功** - 初次尝试失败 → 修正 → 最终成功
2. **复杂问题** - 涉及多个工具/多步骤协作
3. **用户反馈** - 用户明确说"这个解法很好，记下来"
4. **重复模式** - 同一类问题出现3次以上

## 沉淀内容

### 必须记录
- 原始问题描述
- 失败的尝试 & 失败原因
- 修正思路
- 最终解法
- 可复用的模式/代码片段
- 踩过的坑 & 避坑指南

### 可选记录
- 时间消耗（用于评估复杂度）
- 用户的特殊偏好
- 相关的外部资源链接

- 原因: [为什么失败]

```bash
# 具体的命令或代码
```

## 相关技能
- [链接到其他相关技能]

Created: YYYY-MM-DD
Last used: YYYY-MM-DD

```

## 使用流程

1. **解决问题时** - 正常执行，但心里要有"这可能值得记录"的意识
2. **成功后** - 询问用户："这次的解法值得沉淀为技能吗？"
3. **沉淀时** - 创建/更新技能文件，确保格式统一
4. **后续遇到类似问题** - 先检查 skills/learned/ 是否有相关技能
