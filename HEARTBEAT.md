# 🔄 HEARTBEAT - 记忆系统维护

*静默整理机制 - 自动识别和归档重要信息*

---

## 每次会话结束时的检查清单

### 1. 信息评估
检查本次会话是否有值得长期保存的信息：
- [ ] 重要决策或选择
- [ ] 新的用户偏好或习惯
- [ ] 项目进展或状态变更
- [ ] 有效的问题解决模式
- [ ] 用户反馈或纠正
- [ ] 新的联系人信息

### 2. 重要性评分

| 评分 | 标准 | 处理方式 |
|------|------|----------|
| **5分** | 核心身份、关键决策、重要关系 | 立即写入 MEMORY.md 对应章节 |
| **4分** | 项目信息、有效模式、重要偏好 | 写入对应专题文件 |
| **3分** | 有用的背景、临时状态 | 保留在当日日志 |
| **2分** | 一次性信息 | 保留在当日日志 |
| **1分** | 闲聊、临时上下文 | 不记录/清理 |

### 3. 信息归档流程

**如果是 4-5 分信息：**
1. 确定信息类型（决策/偏好/项目/模式/反馈/联系人）
2. 更新对应的专题文件
3. 在 MEMORY.md 的"最近更新"中记录摘要
4. 如有必要，更新 USER.md 或 TOOLS.md

**如果是 2-3 分信息：**
1. 确保记录在当日日志（memory/YYYY-MM-DD.md）
2. 30天后可清理

### 4. 每日日志维护

**每晚检查：**
- [ ] 当日日志是否已创建
- [ ] 重要信息是否已提炼到长期记忆
- [ ] 超过 30 天的日志是否需要清理

### 5. 专题文件同步

**每周检查：**
- [ ] contacts.md - 联系人信息是否最新
- [ ] decisions.md - 是否需要复盘某个决策
- [ ] preferences.md - 是否有新发现的偏好
- [ ] projects.md - 项目状态是否已更新
- [ ] patterns.md - 是否提炼了新的模式
- [ ] feedback.md - 是否有新的反馈需要记录

---

## 触发词识别

当用户在对话中使用以下词语时，自动提升信息重要性：
- "记下来" / "记住这个" → 提升至 4 分
- "别忘了" → 提升至 3 分
- "永久保存" → 提升至 5 分
- "这是一个重点" → 提升至 4 分
- "很重要" → 提升至 4 分

## 记忆检索提醒

每次回复前：
1. 使用 `memory_search` 搜索相关历史记忆
2. 使用 `memory_get` 读取具体片段
3. 在回复中自然引用记忆（如："根据之前的记录..."）
